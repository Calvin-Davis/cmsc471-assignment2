Chicago Crime: Interactive Visualization

dataset

  • the chicago crime dataset from the assignment description; specifically the data from 2024 (need a link to it)
  • chose the data on type of crime and date of crime

rationale for design decisions

  • red bars because crime = violent
  • bar chart because x data is ordinal and y data is ratio data
  • used a categorical color scheme for the pie chart because crime type is nominal and we don't want to imply any order to that
  • added tooltips for the pie chart so that users can see more info about each slice (type it represents, percentage, total count) without cluttering the chart and also encouraging interaction
  • made it so that a clicked bar turns orange, and then the day-level bars that pop up are also orange, so that it encourages the idea that the resulting day-data is based on the clicked month bar (which it is)
  • added functionality and text tip for being able to click on a day-level bar to go back to month level, so users can freely explore and go back without refreshing
  • animated bars and pie slices just because it is engaging and entertaining, as well as being indicative of the interactions

development process

  • code took about 8 hours
  • parts that took the longest:
    • organizing the data and figuring out how to aggregate it, since it's quite a bit of data and somewhat messy. also parsing dates is a little annoying
    • grouping the less common crime types into "other offense" for the pie chart, figuring out best method for it and the logic for it
    • formatting the legend for the pie chart. just getting it to a proper size and good position.
